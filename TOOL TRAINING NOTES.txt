TYPES OF FORMS:-
-----------------
-->Flat forms(eg:-SV001,DM001)
-->Repeating Forms(eg:-AE001)
-->Forms with repeating logs/sections
-->forms with fixed repeating logs/sections

Flat Forms:-
------------
-->It is a single Level form where each field can appear only once for a visit or subject with no repetition.
-->ex:- demographic forms

Repeating forms:-
-----------------
-->A form that can be entered multiple times for the same (visit or subject) to capture the repeated events.
-->example:- adverse events forms

Forms with Repeating Sections:-
-------------------------------
-->A single form that contains a group of fields that can repeat within the form.
-->Ex:- concomitent medications

Forms with Fixed Repeating Sections:-
-------------------------------------
-->A form that contains a fixed number of repeating sections.
-->Ex:- Vital signs forms

Pre-defined Functions in Expression Builder:-
=============================================
1)"abs()":- 
     -->Return the absolute Value of a Number.
     -->Eg:- abs(arg), abs(-4)=4

2)allobjects():-
      -->Return an Array with the Provided Value.
      -->Eg:- allobjects(arg1)

3)array_values():-
     -->Extract value from the elements in the provided Array.
     -->Eg:-  array_values(field,subject|event|instrument)
     -->By default the scope is subject.

4)not():-
    -->It is going to return the boolean value based on the field we provide.
    -->eg:- not(true)=false
            not(false)=true

5)empty():-
    -->Returns true if the provided value (or) array is empty otherwise false.
    -->eg:- empty(654736)=false
            empty([])=true
            empty("")=true

7)completed():-
    -->Returns true if the given array size is greater than zero otherwise return false.
    -->Eg:- completed([1,2,3])=true

8)concatdlm():-
    -->concatenate the given values by using the provided delimiter.
    -->eg:- concatdlm(separator,feild1,field2,field3,..)
            concatdlm("-","one","two")=one-two

9)contains():-
    -->Rerutns true if the provided value contains the second value otherwise return false.
    -->Eg:- contains("abc","bc") = true

10)count():-
    -->Return the Number of elements in the array.
    -->Eg:- count([1,2,3])=3

11)count_values():-
    -->The function counts the number of repetitions of the value in the second argument into an array of values of the first argument.
    -->eg:- count_values(array_values(field),target value)

12)dateadd():-
    -->add or subtract from the date.
    -->eg:- dateadd(date,units,quantity,dateformat(YYYY-MM_DD))

13)datediff():-
    -->Caluculate difference between the dates or times by using the datediff() function.
    -->eg:- datediff(date1,date2,units,returnsignedvalue,decimalvalues)
             units:- weeks,days,months,hours,minutes,seconds

14)startswith():-
     -->Returns true if the first string starts with the second String otherwise return false.
     -->Eg:- startWith("ABCD","AB")

15)endsWith():-
     -->Returns true if the first String ends with the second String otherwise return false.
     -->Eg:- endsWith("ABCD","CD");

16)functions._comparedates():-
      -->compare two dates, if two dates are equal then it will return "0".
      -->If First date is before the Second date then it will return "-1".
      -->If first date is after the second date then it will return "+1".
      -->Eg:- functions._comparedates(date1,date2)

17)functions._getdatedifference():-
      -->Return dates difference between two dates in a specific range.
      -->Eg:- functions._comparedates(date1,date2,"MINUTES"|"HOURS");

18)equals():-
      -->Return true if both the objects are equal.

19)hasall():-
     -->This function compares all elements from the collection are equal to the second argument.
     -->Eg:- hasall([3,3,3])=true

20)hasany():-
     -->This function checks if the first argument(collection) contains the second argument.
     -->eg:- hasany([1,2,3],30)=true.

21)min():-
     -->Return minimum element from the provided array.
     -->Eg:- min([1,2,3])=1

22)max():-
     -->Return maximum element from the provided array.
     -->Eg:-max([1,2,3])=3

23)mean():-
     -->Return avg of the provided array.
     -->Eg:- mean([1,2,3])=2

24)sum():-
     -->Return sum of the elements from the Provided array.
     -->sum([1,2,3])=6.

25)sqrt():-
     -->Returns the perfect positive square root of a number.
     -->sqrt(16)=4

26)indexOf():-
     -->This function returns the index value of a element in an array.
    
27)lenght():-
     -->This function returns length of the provided object.
     -->Eg:- length([1,2,3])=3

28)todecimal() and todouble():-
     -->Returns double value of provided argument.
     -->Eg:- todecimal("2.23")=2.23

29)toint32():-
     -->Returns long value of the provided argument.
    
30)toString():-
     -->return string value of the provided argument.
 
31)isnumeric():-
     -->returns true if the provided argument is numeric.

32)toupper():-

SDV(Soruce Data Verification):-
===============================
-->Soruce Date Verification is a process of checking that the data recorded in trail documents(CRF) matches the source data records or not.
      -->Soruce data= hospital records,blood reports,etc..(CRF)
      -->Trail document(CRF) = where trail data is entered(electronic system).
-->The Main Purpose of SDV is to ensure accuracy,completeness and reliability of trail data.
-->For Example, if the patient blood pressure in hosptial records is entered as 120/80, SDV ensure the same value is entered in the ECRF.
-->Actually Clinical Research Associates will perform this SDV during Monitoring sites.

Dynamic Expressions:-
=====================
-->Dynamic Expressions are used to trigger the events (or) instruments based on certain conditions.
-->There are three types of Dynamic Expressions in RCC they are:
    -->Create Dependent Event
    -->Create Dynamic Event
    -->Add Instrument
-->"Create Dependent Event" is used to trigger the parent and child events based on certain Conditions.
-->"Create Dynamic Event" is used to trigger the Events based on certain Conditions.
-->"Add Instrument" is used to trigger the "n" number of instruments into a particular event or "n" different events based on a certain condition.

      -->In Rules,
            Action TYpe:-
                   -->create dependent Event
                   -->Create Dynamic Event
                   -->Add Instrument

Study Versions and Migration Plans:-
====================================
-->If we Deploy our Package or study  through study version and if we make any updates to that package those updates will not be visible to the old subjects
   they will visible to the new subjects only.
-->If we Deploy our package with the help of Migration plan then whatever the updates we did they will be available to both old and new subjects.
